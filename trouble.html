Solución de posibles inconvenientes


La aplicación web contenida en la página está desarrollada en JavaScript. Concretamente, con herramientas de JS y de JQuery.

Cada navegador o browser actualizado cuenta con JavaScript. Además, cualquiera que ejecute una página web que utilice JQuery automáticamente descarga esta librería (jquery.min.js), permitiendo que pueda cargarse con mayor rapidez en la siguiente ocasión. No obstante, algunas veces esto no se guarda en la memoria por diversos motivos.

Dentro del código HTML, se hace la solicitud de:

"https://ajax.googleapis.com/ajax/libs/jquery/1/jquery.min.js"

Para que el browser integre esta librería debe estar previamente descargada, de lo contrario ocasionará problemas en la página y no se ejecutará correctamente.

Un método para solucionar esta eventualidad es introducir el enlace anterior en el navegador a utilizar con conectividad a internet, previo a la conexión con el ESP8266. El parámetro '/1/' de la cadena solicita la versión más actual disponible.

En algunos casos, esto no soluciona el problema de conexión; por lo que es necesario ajustar parámetros internos del navegador.

Si se está utilizando Google Chrome (probado con versión 76.0.3809.132) u Opera (probado con versión 63.0.3368.94) se puede acceder a la memoria caché de DNS, y así ajustar el valor por defecto. Esto se hace accediendo a la dirección:

"chrome://net-internals/#dns"


En la ventana mostrada en la figura anterior, se debe presionar el botón 'Clear host cache'.

Si el problema de conectividad persiste, es necesario eliminar los valores de cookies, caché y demás datos de navegación y repetir lo explicado al inicio de esta sección (descargar nuevamente la librería jquery.min.js).


¿Cómo saber que es un error de JQuery?


Si al acceder al navegador el usuario puede visualizar el mensaje para ingresar la clave, es un indicio de que el motor Javascript se está ejecutando sin problema. No obstante, si al utilizar las barras deslizadoras de la sección de movimiento en tiempo real estas no presentan ningún cambio y no generan ningún movimiento, es un indicador de que el motor de jQuery no fue cargado correctamente. Para solucionar esto, debe implementar las opciones previamente explicadas o cambiar de navegador. Desde un celular, se recomienda utilizar Mozilla Firefox, ya que presenta más estabilidad para la descarga de jQuery.


¿El problema no está descrito en las secciones anteriores?


Si el usuario comprueba que todo está configurado correctamente pero presenta dificultades de conectividad puede deberse a:


1- El dispositivo ESP8266 se encuentra saturado y no emite la señal WiFi.

2- El dispositivo ESP8266 no fue acoplado correctamente al brazo robot.



En cuanto al caso 1:

Puede deberse a que el dispositivo fue reiniciado varias veces estando activo el puerto serial, lo que ocasiona un comportamiento no deseado en el microcontrolador e impide emitir la señal WiFi. Para solucionar esto, es necesario desacoplar ambos microcontroladores y reiniciar ambas conexiones. Una vez hecho esto, el problema quedará solventado.

Respecto al caso 2:

Esto suele ocurrir cuando el dispositivo ESP8266 se enciende luego de conectar el brazo robot, ocasionando que no exista un acople en la transmisión serial de parte del microcontrolador dsPIC30F3011. Para solucionar esto, se recomienda encender primero el módulo ESP8266, esperar la señal WiFi y seguidamente encender el brazo robot.
